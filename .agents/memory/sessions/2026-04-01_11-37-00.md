## Key Decisions Made

### Architecture: Single Entry Point via Nginx
- All traffic → Nginx on port 80
- Nginx serves React SPA static build
- Nginx proxies `/api/*` → backend:5000 (internal, not exposed to host)
- Frontend uses relative paths (`/api/*`), zero CORS configuration needed

**Why**: Eliminates CORS complexity, provides clean single entry point, enables network isolation

### API URL Strategy
- Frontend axios baseURL hardcoded as `/api` (relative paths)
- VITE_API_URL accepted in docker-compose but unused (future flexibility)
- This works because Nginx proxy makes backend accessible at same domain/port

**Why**: No environment variable injection needed at runtime, cleaner than absolute URLs

### SQLite Persistence
- Named Docker volume `db_data` mounted at `/app/data`
- Alternative (bind mount) rejected due to portability issues
- Survives container restart, can be backed up with `docker volume`

### Health Check
- Backend must pass healthcheck before frontend starts
- 40s startup grace period (migrations take time)
- Prevents Nginx serving broken app
