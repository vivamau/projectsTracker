## Architecture (Free Tier)

```
Browser
  ↓ HTTPS
Nginx Frontend (projectstracker-frontend.onrender.com)
  ├─ Serves React SPA
  └─ Proxies /api/* → backend (via BACKEND_URL env var)
       ↓ HTTPS
Express Backend (projectstracker-api.onrender.com)
  ├─ API endpoints
  └─ SQLite in /tmp (temporary, resets on restart)
```

## Free Tier Characteristics

### What Works ✅
- Full Docker deployment
- GitHub auto-deploy (push → auto-redeploy)
- Free HTTPS/SSL certificates
- Public URLs (xxx.onrender.com)
- Testing and demos
- Easy upgrade path to paid tier

### What Doesn't Work ❌
- No persistent disk (data in /tmp resets on restart)
- Service sleep after 15 min idle (free tier limitation)
- First request after sleep is slow (~20-30s)
- No custom domains
- No production-ready (data loss risk)

### When to Upgrade to Starter
- Cost: $7/service/month ($14/month total)
- Benefit: Persistent 1GB disk + always-on service
- Upgrade when: Need persistent data, 24/7 uptime, or real users

## Key Design Decisions

### 1. Hardcoded URLs Over fromService
- Render's fromService doesn't support `hostWithScheme`
- Valid properties: host, hostport, port, connectionString
- Solution: Hardcoded full HTTPS URLs (deterministic in Render)
- Trade-off: If service names change, must update URLs manually

### 2. Free Tier Approach Over Blueprint
- Blueprint is paid feature (not in free tier)
- Manual service creation is free
- Trade-off: 10 extra clicks, but full documentation provided
- User can upgrade to Blueprint later if desired

### 3. /tmp Database Location
- Free tier can't use /app/data (would require persistent disk)
- SQLite goes to /tmp (temporary storage)
- Data resets on restart (expected for free tier)
- Clear documentation warns about this limitation
- Upgrade path available when ready for persistence

### 4. Separated Documentation
- Created render_com/ folder (not polluting root)
- 7 files organized by use case (overview, setup, troubleshooting, upgrade)
- User can follow a clear reading order
- Quick reference available for experienced users
