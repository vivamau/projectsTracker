Access Control

Four roles. One source of truth.

A layered permission model ensures the right people have the right access — enforced server-side on every request.

superadmin
Super Admin

Full system access including user management, audit logs, and application settings.

  • User CRUD
  • Audit log access
  • App settings
  • All data operations
admin
Admin

Full access to project data and content. Cannot manage users or view audit logs.

  • Create & edit projects
  • Manage vendors
  • Budget management
  • Division management
contributor
Contributor

Read-only by default. Full edit rights on projects where assigned as PM or Solution Architect.

  • View all content
  • Edit own projects
  • Manage milestones
  • Record consumption
guest
Guest

Read-only access across the entire platform. Ideal for stakeholders and observers.

  • View projects
  • View budgets
  • View vendors
  • View initiatives
Quickstart

Up and running in minutes.

Docker is the recommended path for deployment. For local development, run backend and frontend independently.

01

Clone & configure

Clone the repository and copy the environment file. Edit .env to set your secrets and ports.

02

Build & launch

Run docker compose up --build -d. The backend auto-runs migrations, seeds roles, users, and sample data.

03

Sign in

Navigate to http://localhost and log in with admin@projecttracker.it.

04

Explore the API docs

Swagger UI is available at /api/docs with all endpoints documented and interactive.

terminal
# Clone the repo
git clone https://github.com/vivamau/projectsTracker
cd projectsTracker

# Configure environment
cp .env.docker.sample .env

# Build and start with Docker
docker compose up --build -d

# Or run locally (dev mode)
cd backend && npm install && npm start
cd frontend && npm install && npm run dev

# Run the test suite
cd backend && npm test
cd frontend && npm run test:e2e
